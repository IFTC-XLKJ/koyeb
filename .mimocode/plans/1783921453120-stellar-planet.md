# 优化反爬机制 - server.ts

## 目标

增强 `server.ts` 中的反爬虫机制，提升对自动化爬虫、扫描工具和恶意请求的防护能力。

## 当前机制分析

| 机制 | 位置 | 现状 |
|------|------|------|
| UA 黑名单 | L31-53 | 硬编码 22 个已知爬虫，覆盖不全 |
| 空 UA 拦截 | L227-228 | 仅检查 null |
| 可疑行为检测 | L67-76 | 仅检查 3 个 header，过于宽松 |
| 速率限制 | L78-89 | 100 次/分钟/IP，无封禁机制 |
| UA 精确匹配 | L238-241 | 仅匹配 "Mozilla/5.0" |

## 改进方案

### 1. 增强 UA 黑名单 (`crawlerAgents`)

**文件**: `server.ts` L31-53

添加更多已知爬虫和自动化工具：
```
新增:
- 搜索引擎: mj12bot, ahrefsbot, semrushbot, yandexbot, sogou, exabot, dotbot, bytespider, gptbot, chatgpt-user, ccbot
- 爬虫框架: httplib, wget, httpx, aiohttp, fetch, clawer, crawler
- 自动化工具: headlesschrome, phantomjs, slimerjs, cypress, nightwatch
- 监控工具: uptimebot, pingdom, statuspage, healthcheck
- AI 训练: ccbot, facebookbot, googlebot (区分)
```

### 2. 新增子串模式匹配

**文件**: `server.ts` - 新增 `uaSubstrings` 数组

将精确匹配改为包含子串匹配，覆盖变体 UA：
```typescript
const uaSubstrings = [
    "bot", "spider", "crawl", "fetch", "scrape", "harvest",
    "headless", "phantom", "selenium", "webdriver", "automation",
    "httpclient", "python-requests", "python-urllib",
    "java/", "go-http", "php/", "ruby/", "perl/",
    "curl/", "wget/", "libwww", "lwp-trivial",
];
```

### 3. 改进可疑行为检测 (`isSuspiciousBehavior`)

**文件**: `server.ts` L67-76

增强检测维度：
- UA 长度 > 500 字符 → 可疑
- 缺少 `sec-fetch-dest` / `sec-fetch-mode` → 可疑（现代浏览器必带）
- UA 不含 "Mozilla"、"Chrome"、"Firefox"、"Safari" 任一关键词 → 可疑
- 缺少 `accept-language` 且 `accept` 为空 → 可疑

### 4. 新增 IP 封禁机制

**文件**: `server.ts` - 新增 `bannedIPs` Map

```typescript
const bannedIPs = new Map<string, number>(); // ip -> banExpiresAt

function isIPBanned(ip: string): boolean {
    const expires = bannedIPs.get(ip);
    if (!expires) return false;
    if (Date.now() > expires) {
        bannedIPs.delete(ip);
        return false;
    }
    return true;
}

function banIP(ip: string, durationMs: number): void {
    bannedIPs.set(ip, Date.now() + durationMs);
}

function recordViolation(ip: string): void {

}
```

### 5. 速率限制分层

**文件**: `server.ts` L78-89

- 普通页面/API: 100 次/分钟
- 认证相关 (`/api/user/login`, `/api/user/register`): 10 次/分钟
- 敏感操作: 5 次/分钟

### 6. URL 模式拦截

**文件**: `server.ts` onRequest hook

在 PHP 拦截后新增：
```typescript
// 阻止常见攻击路径
const blockedPaths = [
    "/wp-admin", "/wp-login", "/xmlrpc.php",
    "/.env", "/.git", "/config.php",
    "/phpmyadmin", "/admin", "/backup",
    "/cgi-bin", "/scripts",
];
if (blockedPaths.some(p => request.url.toLowerCase().startsWith(p))) {
    return reply.status(404).send({ code: 404, msg: "Not Found", timestamp: time() });
}
```

### 7. 日志优化

**文件**: `server.ts` L254

移除 `console.log("Cookies:", request.headers.cookie)` — 日志中打印 Cookie 有安全风险且产生噪音。

## 实施步骤

1. 扩展 `crawlerAgents` Set，添加新条目
2. 新增 `uaSubstrings` 数组（子串匹配）
3. 新增 `blockedPaths` 数组
4. 改进 `isSuspiciousBehavior` 函数
5. 新增 `bannedIPs` Map 及相关函数（`isIPBanned`, `banIP`, `recordViolation`）
6. 修改 `isRateLimited` 支持分层限制
7. 修改 onRequest hook：先检查封禁 → 再检查子串 → 再检查速率 → 再检查可疑行为
8. 移除 Cookie 日志打印
9. 新增定期清理 `bannedIPs` 的 `setInterval`

## 验证方式

1. 启动服务器：`bun dev` 或 `npx tsx server.ts`
2. 测试正常浏览器访问 → 应正常通过
3. 测试空 UA 请求 → 应返回 400
4. 测试已知爬虫 UA（如 `python-requests/2.28.0`）→ 应返回 403
5. 测试 `/wp-admin` 路径 → 应返回 404
6. 测试速率限制 → 超过阈值应返回 429
